# Installation and environment management

## Supported setup

The project is developed with Python 3.11 and PyTorch 2.3. It is currently
verified in the Conda environment named `rgc_subunit_classifier` on macOS. The
numerical code is platform independent, but PyTorch CPU/GPU packages differ by
operating system and accelerator.

## Recommended: create the Conda environment

From the repository root:

```bash
conda env create -f environment.yaml
conda activate rgc_subunit_classifier
python -m pip install -e .
python -m pytest
```

`environment.yaml` intentionally contains only Python/pip at the Conda level and
the direct project packages in its pip section. This reflects how the existing
environment was built and avoids exporting platform-specific transitive Conda
builds. `pip` resolves the transitive dependencies appropriate to the machine.

The editable installation means changes under `src/` are immediately available
without reinstalling the package.

## Jupyter kernel

Register the environment once:

```bash
python -m ipykernel install --user \
  --name rgc_subunit_classifier \
  --display-name "Python (rgc_subunit_classifier)"
```

Then launch from the repository root:

```bash
jupyter lab
```

Select `Python (rgc_subunit_classifier)` in each tutorial notebook.

## Pip-only installation

If Conda is unavailable:

```bash
python3.11 -m venv .venv
source .venv/bin/activate             # Windows: .venv\Scripts\activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python -m pip install -e .
python -m pytest
```

This is supported for convenience, but `environment.yaml` is the canonical
environment description.

## Existing environment

To update an existing environment from the YAML without recreating it:

```bash
conda activate rgc_subunit_classifier
conda env update -f environment.yaml --prune
python -m pip install -e .
python -m pytest
```

Review `--prune` before using it in an environment that contains unrelated
packages: it removes packages not required by the specification.

## Dependency policy

| File | Purpose |
|---|---|
| `environment.yaml` | Recreate the complete analysis, tests, and notebooks environment |
| `requirements.txt` | Pinned direct pip dependencies equivalent to the YAML pip section |
| `pyproject.toml` | Runtime metadata for installing this repository as a package |

Runtime dependencies are kept broader in `pyproject.toml` so the package can be
installed into compatible environments. The environment and requirements files
pin the versions used for reproduction.

## PyTorch and accelerators

The specification installs the standard PyPI PyTorch wheel. If CUDA or another
platform-specific build is needed:

1. Create the environment.
2. Remove the existing PyTorch build if necessary.
3. Use the command generated by the official PyTorch installation selector.
4. Run the test suite.

Do not copy a CUDA installation command between machines without checking driver
and toolkit compatibility.

The pipelines choose CUDA when `torch.cuda.is_available()` is true, otherwise
CPU. Different devices can produce small floating-point or ordering differences.

## Optional STNMF baseline

`src.models.stnmf_baseline` is an adapter, not a vendored implementation. The
historical Colab archive included a copied third-party package; it is deliberately
excluded from the maintained environment. Install a maintained compatible
`stnmf` distribution separately if this comparison is required, then verify the
adapter against a known result before reporting it.

## Environment checks

```bash
python --version
python -c "import h5py, matplotlib, numpy, scipy, sklearn, shapely, torch"
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
python -m pytest
```

To inspect direct and transitive packages:

```bash
conda list -n rgc_subunit_classifier
python -m pip check
```

## Common problems

### Environment exists already

```bash
conda env update -n rgc_subunit_classifier -f environment.yaml
```

Or remove and recreate it only if no irreplaceable local packages are present:

```bash
conda env remove -n rgc_subunit_classifier
conda env create -f environment.yaml
```

### Project imports fail

Confirm the editable install and working directory:

```bash
pwd
python -m pip show rgc-subunit-classifier
python -c "from src.simulation.run_pipeline import run_subunit_model"
```

### Matplotlib cannot write its cache

```bash
export MPLCONFIGDIR=/tmp/matplotlib-$USER
mkdir -p "$MPLCONFIGDIR"
```

### Jupyter uses another Python

Inside a notebook:

```python
import sys
print(sys.executable)
```

It should point inside the `rgc_subunit_classifier` environment.
